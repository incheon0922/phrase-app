# 成语词汇积累小程序 UI 设计稿

本设计稿基于当前需求输出，目标是先确定产品的视觉方向与页面结构，便于后续直接进入前端开发。

适用范围：

1. 首页
2. 练习页
3. 我的页面
4. 导入入口与空状态

## 1. 设计定位

产品关键词：

1. 专注
2. 清晰
3. 轻学习感
4. 备考工具属性

整体风格建议：

1. 以浅色为主，避免过度娱乐化。
2. 视觉上偏“学习卡片 + 题库练习”。
3. 突出可读性，弱化装饰噪音。
4. 通过少量暖色强调练习和进度反馈。

适合的用户感受：

1. 打开就能开始学
2. 看起来有秩序，不乱
3. 重点功能一眼可见
4. 做题反馈直接明确

## 2. 视觉风格建议

### 2.1 色彩系统

建议以米白、深墨绿、暖橙、成功绿、错误红为主。

```txt
主背景色:   #F7F3EA
卡片底色:   #FFFDF8
主文字色:   #2B2B2B
次文字色:   #7A746B
主品牌色:   #2F5D50
强调色:     #D98F39
成功色:     #2E9B63
错误色:     #D94F45
边框色:     #E7DFD1
禁用色:     #C8C1B6
```

颜色逻辑：

1. `主品牌色` 用于核心按钮、导航选中态。
2. `强调色` 用于“开始练习”“今日目标”等吸引注意的区域。
3. `成功色` 和 `错误色` 只用于答题反馈，避免滥用。

### 2.2 字体建议

小程序里建议保持系统兼容，但在视觉语气上走“书卷感 + 工具感”。

字体层级建议：

1. 页面大标题：`34rpx`，中黑
2. 模块标题：`30rpx`，中黑
3. 正文：`28rpx`
4. 说明文字：`24rpx`
5. 题目正文：`32rpx - 36rpx`

### 2.3 圆角与阴影

```txt
大卡片圆角: 28rpx
中卡片圆角: 22rpx
按钮圆角:   18rpx
选项圆角:   20rpx
阴影:       0 10rpx 30rpx rgba(61, 50, 33, 0.08)
```

## 3. 全局布局规范

### 3.1 页面安全边距

模块上下间距: 24rpx

## 4. 信息架构

底部导航：

推荐补充的隐藏入口：

2. 首页卡片中的“错题本”

## 5. 首页设计稿

首页的核心目标是：

1. 让用户快速开始练习
2. 让用户知道自己今天该做什么
3. 让用户看到自己的词汇积累状态

### 5.1 首页模块结构

1. 顶部品牌欢迎区
2. 今日练习主卡
3. 快捷功能区
4. 词库概览区
5. 最近学习记录

### 5.3 首页视觉说明

顶部欢迎区建议：

1. 使用浅米色到暖白色渐变背景。
2. 右上角可放连续学习天数徽标。
3. 标题使用更大的字号和更稳重的字重。

“开始练习”按钮建议：

1. 大尺寸主按钮，放在今日练习卡正中。
2. 用 `主品牌色` 或 `强调色` 做填充。
3. 文案可以为 `开始练习` 或 `继续今日练习`。

快捷区建议：

1. 两个半宽卡片并排。
2. 左边固定是“导入词汇”，右边固定是“错题本”。
3. 方便用户快速进入高频功能。

## 6. 练习页设计稿

练习页是全产品最重要的页面，重点要保证：

1. 题干突出
2. 选项易点
3. 正误反馈明确
4. 节奏顺畅

### 6.1 练习页结构

1. 顶部进度栏
2. 题目卡片
3. 四个选项按钮
4. 作答反馈区
5. 下一题按钮

### 6.4 练习交互建议

作答流程：

1. 点击任一选项后立即锁定本题。
2. 若答对，仅当前选项变绿。
3. 若答错，当前选项变红，正确项变绿。
4. 解析区从折叠态展开。
5. `下一题` 按钮变为可点击。

建议补充的小体验：

1. 答对时轻微成功动画。
2. 答错时短暂震动反馈。
3. 连续答对可展示“已连续答对 5 题”。

## 9. 空状态设计

这是首版很容易忽略但非常重要的部分。

### 9.1 首页空状态

适用场景：

1. 用户还没有任何词条

建议样式：

```txt
插画/图形占位
还没有词汇内容
先导入你的第一批成语词汇吧
[ 导入词汇 ]
[ 手动新增 ]
```

### 9.2 错题本空状态

```txt
今天还没有错题
继续保持，去练两题试试
[ 开始练习 ]
```

## 10. 图标建议

建议图标风格：

1. 线性图标
2. 略带圆角
3. 保持统一笔画粗细

推荐图标语义：

1. 首页：书页 / 屋檐
2. 我的：用户轮廓
3. 导入：向内箭头 / 文档
4. 错题本：书签 / 警示圆点
5. 收藏：星标

## 11. 动效建议

动效不要多，但要有记忆点。

建议保留以下 3 类：

1. 页面加载时卡片自下而上淡入
2. 点击开始练习按钮时轻微缩放反馈
3. 题目答对后出现短暂高亮过渡

建议时长：

```txt
页面入场: 300ms
按钮点击: 160ms
答题反馈: 220ms
```

## 12. 适配建议

小程序页面需要优先考虑：

1. 顶部安全区
2. 底部安全区
3. 小屏幕机型的内容折叠

注意点：

1. 首页卡片不要堆太多，避免首屏过长。
2. 练习页的四个选项按钮要保证足够高，方便点击。
3. 解析区默认不要太高，避免遮挡下一题按钮。

## 14. 首版 UI 落地建议

如果按最小成本先做一版，建议你先实现这三个页面：

1. 首页
2. 练习页
3. 我的页面

优先保留这些视觉重点：

1. 首页一个强主按钮
2. 练习页一个强题目卡
3. 正误状态颜色明确
4. 底部导航简洁稳定

## 15. 设计结论

这版 UI 建议整体走：

1. 浅底色
2. 卡片化布局
3. 绿色为主品牌色
4. 暖橙做练习强调
5. 成功绿 / 错误红做答题反馈

这样既有学习产品的沉稳感，也不会像传统题库那样太生硬。

如果继续往下做，下一步最合适的是直接把这版设计稿转成页面代码，并先完成：

1. 首页 UI
2. 练习页 UI
3. 我的页面 UI
4. TabBar
